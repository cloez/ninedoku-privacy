# Ninedoku — 데이터 안전 양식 (Play Console 입력용)

**작성일**: 2026-06-18
**앱**: Ninedoku (com.cloez.sudoku)

---

## Play Console "데이터 안전" 섹션 답변

### 1. 데이터 수집 및 공유 개요

**Q: 앱이 필수 사용자 데이터 유형을 수집하거나 공유합니까?**
→ **아니요**

**근거**: Ninedoku는 인터넷 권한 자체가 없으므로 (AndroidManifest.xml에서 `tools:node="remove"`로 명시 제거) 어떤 데이터도 외부로 전송할 수 없습니다. 광고 SDK, 분석 SDK, 크래시 리포팅 도구가 전혀 포함되어 있지 않습니다.

### 3. 데이터 보안

**Q: 앱에서 수집하는 모든 사용자 데이터가 전송 중에 암호화됩니까?**
→ **해당 없음** (네트워크 통신 자체가 없음)

**Q: 사용자가 데이터 삭제를 요청할 수 있는 방법을 제공합니까?**
→ **해당 없음** (수집 데이터 없음)

**참고**: 앱 삭제 시 모든 로컬 데이터(SharedPreferences, 로컬 파일)가 Android OS에 의해 자동 삭제됩니다.

---

### 4. 데이터 처리 관행 요약

```
본 앱은 사용자 데이터를 수집하지 않습니다.

• 인터넷 권한 없음 — 네트워크 접근 자체가 불가능
• 광고 SDK 없음 — 광고 ID 미사용
• 분석 SDK 없음 — 사용 패턴 추적 없음
• 크래시 리포팅 없음 — 오류 보고 외부 전송 없음
• 제3자 SDK 미사용 — 데이터 수집 라이브러리 없음

모든 게임 데이터(진행률, 통계, 설정, 배지)는 기기 로컬에만 저장되며,
앱 삭제 시 자동으로 완전 삭제됩니다.
```

---

### 5. Play Console 입력 절차

1. Play Console → 앱 → **앱 콘텐츠** → **데이터 안전**
2. "시작하기" 클릭
3. "앱에서 필수 사용자 데이터 유형을 수집하거나 공유합니까?" → **아니요** 선택
4. 저장 및 제출

---

### 6. 기술 근거 (검수 대비)

| 검증 항목 | 코드 위치 | 결과 |
|-----------|-----------|------|
| INTERNET 권한 제거 | `AndroidManifest.xml` L4 | `tools:node="remove"` |
| 광고 SDK 부재 | `pubspec.yaml` 의존성 | google_mobile_ads 없음 |
| 분석 SDK 부재 | `pubspec.yaml` 의존성 | firebase/sentry/amplitude 없음 |
| 네트워크 라이브러리 부재 | `pubspec.yaml` 의존성 | http/dio 없음 |
| HTTP 호출 부재 | `lib/` 전체 grep | import 0건 |
