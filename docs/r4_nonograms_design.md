## STEP 1: GD 기획서

### 1. 게임 개요
행/열에 주어진 숫자 힌트를 보고 격자를 채워 그림을 완성하는 퍼즐.
숫자 힌트는 해당 행/열에서 연속으로 채워지는 셀의 개수를 나타낸다.

### 2. 규칙 정의
1. 각 행/열 옆의 숫자 리스트는 연속으로 채워야 하는 셀 그룹의 크기를 순서대로 나타냄
2. 그룹 사이에는 최소 1칸의 빈칸이 있어야 함
3. 숫자 리스트가 정확히 만족되도록 셀을 채우면 승리
4. X 표시로 "이 셀은 비어있다"를 표시할 수 있음 (선택적)

#### 규칙 설명 텍스트
```
1. 행/열의 숫자는 연속으로 채울 셀의 개수입니다
2. 숫자 사이에는 최소 1칸의 빈칸이 필요합니다
3. 모든 행과 열의 숫자 조건을 만족시키면 완성입니다
```

### 3. 난이도 체계
| 난이도 | 격자 | 특성 |
|--------|------|------|
| 입문 | 5×5 | 단순 패턴, 힌트 큼 |
| 쉬움 | 10×10 | 기본 교차 논리 |
| 보통 | 15×15 | 복합 패턴 |
| 어려움 | 20×20 | 고급 추론 |

### 4. 등급 기준
| 난이도 | 기준 시간 |
|--------|----------|
| 입문 | 120초 |
| 쉬움 | 360초 |
| 보통 | 900초 |
| 어려움 | 1800초 |

### 5. 모드

### 7. 힌트 4단계
- Level 1: "행 {R}을 살펴보세요"
- Level 2: "이 행의 힌트 [{n1}, {n2}]를 보면, 일부 셀을 확정할 수 있습니다"
- Level 3: "힌트 합계 + 간격을 계산하면, 겹치는 영역이 반드시 채워져야 합니다"
- Level 4: 정답 자동 채우기

### 8. 고유 설정
- 자동 X 표시: 행/열 완성 시 나머지 빈칸에 자동 X

## STEP 2: UX 명세서

### 인터랙션: 패턴 C (노노그램 전용)
```
셀 탭 → 채움(■)
길게 누르기 → X 표시
드래그 → 연속 채움/X
```

### 입력 모드 컨트롤 바
```
[■ 채우기] [✕ 지우기] [↩ 되돌리기]
```

### 보드 레이아웃 (세로 모드)
```
┌─────────────────────┐
│ AppBar (노노그램)     │
│ 타이머 | 실수 | 힌트   │
├───┬─────────────────┤
│   │ 열 힌트 영역      │
│행 ├─────────────────┤
│힌 │                 │
│트 │   게임 보드       │
│   │                 │
├───┴─────────────────┤
│ 힌트 메시지           │ ← AnimatedSize
├─────────────────────┤
│ [■채우기] [✕] [↩]    │
│ [💡힌트]              │
└─────────────────────┘
```

### 셀 렌더링
- 채워진 셀: 검은 사각형(■) 채움
- X 셀: X 마크
- 빈 셀: 비어있음
- 완성된 행/열: 힌트 숫자 색상 변경 (회색 → 초록)
