## C. 챔피언 관점 가치 평가

### C.1 입문자 (스도쿠 1~10판 경험)

- **충분한가?** **부분적으로 그렇다.** L1 영역 펄스 + "이 박스를 살펴보세요" 메시지는 입문자가 시야를 좁히는 데 즉시 효과적이다. L2 기법 이름 노출도 학습 동기를 자극한다.
- **부족한 점**: "Naked Single — 후보가 하나만 남았어요"라는 L2 메시지를 읽어도, 입문자는 **어느 9개 숫자가 막혔는지** 시각으로 확인할 수단이 없다. 결국 L3 텍스트를 읽고 머릿속으로 추론해야 하는데, 입문자에게는 인지부하가 높다.
- **권고**: L2 또는 L3에서 **셀 내부에 후보 숫자 미니 메모를 자동 표시**하는 것이 결정타가 될 것이다. 이게 입문자 학습 가치의 70%를 차지한다.

### C.2 중급자 (Hidden Single, Naked Pair 사용자)

- **충분한가?** **거의 그렇다.** L2의 기법 이름 노출만으로도 "아, 여기서 Hidden Pair를 찾으면 되겠구나"라는 힌트를 얻는다. 영역 펄스로 어디인지도 안다.
- **부족한 점**: Naked Pair의 경우 "두 셀"이 어디인지 강조가 없다. 박스 전체 펄스만으로는 페어가 무엇인지 모른다.
- **권고**: L2에서 **기법의 핵심 셀(2~4개) 테두리 강조**가 가장 가성비 높다. 이 한 가지만 추가해도 중급자에게는 합격선.

### C.3 고급자 (X-Wing 이상 사용자)

- **충분한가?** **아니다.** X-Wing은 4개 셀이 사각형을 이뤄야 시각적으로 이해되는데, 박스 펄스만으로는 X-Wing 구조 자체를 보여주지 못한다.
- **단**, 고급자는 어차피 힌트를 거의 안 쓴다. 따라서 우선순위 낮음.
- **권고**: 다음 사이클 P1으로 "X-Wing 사각형 4셀 강조 + 소거 라인" 시각화. 학습 모드의 부산물로 만들면 효율적.

### C.4 단계 차별화 — 특히 L2 vs L3

이번 분기 작업에서 가장 큰 개선. `hint.l2.*` vs `hint.l3.*` 메시지가 명확히 다르다:

- **L2 (소개)**: "이 영역에서 숫자 {n}이(가) 들어갈 수 있는 칸은 한 곳뿐이에요" — *what*
- **L3 (이유)**: "같은 영역의 다른 칸들에는 숫자 {n}이(가) 들어갈 수 없어요. 그래서 이 칸만 가능해요" — *why*

**평가**: 텍스트만 본다면 차별화 OK (사용자 보고 이슈 2 해결). 그러나 시각으로 보면 L2와 L3가 동일하다 (둘 다 박스 펄스 후 정적). 결국 사용자는 "글자만 좀 바뀌었네" 정도로 인식할 위험이 여전히 30% 남았다. → **L3에 소거 후보 X 표시가 차별화의 핵심**.

### C.5 L4 대신 한 단계 추가 가치 (예: "메모 모드 시도해보세요")

**부정적 평가**. 4단계는 이미 인지부하가 높다. 5단계는 사용자가 "도대체 정답을 언제 보여주는 거야"라며 좌절할 위험. 대신 다음을 권고:

- **L3 강화안**: L3 진입 시 **자동으로 메모 후보를 셀에 채워준다** (사용자 명시 메모와 시각적으로 구분, 예: 회색 작은 숫자). 사용자가 직접 메모 모드를 켤 필요 없이, L3가 곧 "후보 시각화 + 이유 설명"이 되도록.
- 이 방식이 5단계 추가보다 훨씬 낫다.

## D. 보완 권고 (P0/P1/P2)

### D.1 P0 — 이번 사이클 마저 끝내야 할 것

#### D.1.1 L2 관련 셀 초록 테두리 (가치 高 / 공수 L)
- **이유**: 1차 리뷰 P0 #5의 미충족 부분. Naked Pair, Hidden Pair, Pointing Pair 같은 다중 셀 기법에서 "어디 두 셀인지" 시각화 부재가 가장 큰 갭.
- **사양**: `HintResult.highlightCells`(또는 신규 `keyCells: List<(int,int)>` 필드) 활용. `sudoku_board_widget.dart`의 `_CellWidget`에서 `level == 2 && keyCells.contains((row,col))`이면 테두리 `#10B981` 2.5px.
- **공수**: L (2~3시간). 필드 추가 + 위젯 분기 + 테스트.

#### D.1.2 L3 후보 숫자 자동 메모 시각화 (가치 高 / 공수 M)
- **이유**: 입문자 학습 가치의 결정타. "왜 5인가"를 텍스트로 읽는 것보다 셀 안에 1,2,4,6,7,8,9가 빨간 사선으로, 3,5가 초록으로 보이는 게 100배 효과적.
- **사양**: L3 진입 시 대상 셀 + 같은 영역 빈 셀들에 후보 메모 자동 렌더 (사용자 메모와 다른 색: 회색 8sp). 소거되는 숫자에 빨간 사선.
- **공수**: M (4~6시간). `HintResult.eliminations` 필드는 이미 1차 리뷰 사양에 포함, 위젯 렌더링이 본체.

### D.2 P1 — 다음 사이클

#### D.2.1 L4 대상 셀 글로우 + 숫자 scale 애니메이션 (가치 中 / 공수 L)
- **이유**: 현재 L4는 "갑자기 숫자가 박힌다"는 무미건조한 경험. 글로우 0.8초 + scale 0.5→1.2→1.0 (400ms ease-out-back) 추가 시 만족감 큼.
- **공수**: L (1~2시간). `AnimatedSwitcher` + `TweenAnimationBuilder`만으로 충분.

#### D.2.2 hintCount 정책 — L1 무료화 검토 (가치 中 / 합의 필요)
- **현재**: L1 진입 시 +1, L4 정답 시 +1, 총 +2.
- **권고안**: L1 무료, L2 진입 시 +1, L4 +1, 총 +2 동일. 사용자가 영역만 보고 스스로 풀면 0 카운트.
- **합의 필요**: GD + GC + 사용자. 등급 임계값 영향 없음(총합은 동일).
- **공수**: S (30분 코드, 의사결정 시간이 본체).

#### D.2.3 텍스트 톤 — "이에요/어요" 일관성 점검 (가치 低 / 공수 L)
- 현재 메시지 톤이 친근체("~이에요/어요")로 통일된 점은 우수. 다만 영어 번역에서 직역체 ("This box already contains...")로 빠지지 않도록 4언어 톤 가이드 1페이지 정리 권고.
- 공수: L (30분).

### D.3 P2 — 백로그

### D.4 단계 통합/분리 — 4단계가 적정한가

**결론: 적정하다.** 4단계 유지 권고.

- 3단계로 축소 시: L1+L2 통합하면 "영역 + 기법명"이 한 번에 노출되어 학습 점진성 손실.
- 5단계로 확대 시: 인지부하 증가, "정답이 언제 나오나" 좌절.
- **현재 구조의 강점**: L1(어디) → L2(무엇) → L3(왜) → L4(정답) 4단계가 인지 단계와 정확히 매핑.

## 결론

1차 리뷰 시점(23/50) 대비 **+13점 도약(36/50)**으로 목표(38)에 -2 근접. **다국어 키 시스템과 HintBanner 통합, 96px 고정 슬롯, L2/L3 메시지 분기는 챔피언 기준에서 충분히 우수**하다. 실기기 이슈 3건(흔들림/메시지 중복/포커스 잔존)도 모두 해소되어 v1.0.1+4는 **출시 가능한 수준**이다. 다만 1차 리뷰 P0의 핵심인 "왜 그 답인지 시각으로 증명"이 여전히 부재해, 입문자 학습 가치가 50% 수준에 머문다. **다음 사이클에 L3 후보 자동 메모 시각화(M) + L2 관련 셀 테두리(L) 두 가지만 더 추가하면 44/50로 Sudoku.com을 추월**한다. 추가 작업의 ROI가 매우 높으므로 권고한다.
