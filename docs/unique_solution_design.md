## 1. 문제 정의

퍼즐 게임의 핵심 품질 지표는 **유일해(unique solution)**다.
유일해가 보장되지 않으면:
- 사용자가 논리적으로 풀었는데 게임이 완료되지 않음 (등록된 정답과 불일치)
- 또는 여러 정답이 존재하여 풀이가 추측이 됨 ("아무거나 맞으면 OK")
- 두 경우 모두 게임의 신뢰도를 치명적으로 떨어뜨림

R12까지 13게임 출시 후 전문가 평가에서 4개 게임이 유일해 보장 미흡으로 판정됨.

## 2. 개선안

### 2.1 killer_sudoku — 유일해 검증 추가
- **현재**: 케이지 분할 + 완성 보드 생성 후 즉시 반환, 유일해 검증 없음.
- **변경**: solver를 모드 변경 — 두 번째 해를 찾는 즉시 중단하는 `countSolutions(maxCount: 2)` 추가.
- 두 번째 해가 발견되면 그 케이지 분할은 폐기하고 재시도.

### 2.2 yin_yang — master 크기 조정 + 검증 확장
- **현재**: master = 20×20, hard/master는 유일해 검증 스킵.
- **변경**: 
  - master 사이즈 20 → 16으로 축소 (검증 비용 약 75% 절감)
  - 모든 사이즈에서 유일해 검증 활성화

### 2.3 tents — master 크기 조정 + 검증 확장
- **현재**: master = 14×14, hard/master는 유일해 검증 스킵.
- **변경**:
  - master 사이즈 14 → 12로 축소
  - 모든 사이즈에서 유일해 검증 활성화

### 2.4 nonograms — 유일해 검증 강화
- **현재**: solver가 첫 답을 찾아 원본과 비교만. 다른 답 존재 여부 미확인.
- **변경**:
  - `solveCount(board, maxCount: 2)` 추가
  - 2개 이상 답이 있으면 그 퍼즐은 폐기하고 재시도

## 3. 비기능 요구사항

- 모든 변경 후에도 **생성 시간 3초 이내 보장**.
- 시간 초과 시 폴백: 더 큰 시드 범위로 재시도.
- 기존 저장된 게임(SharedPreferences)에 영향 없도록 backward compatible.
- 난이도 변경(yin_yang master 20→16, tents master 14→12)은 **사용자 영향 명시 필요** 없음. 첫 출시이므로 기존 사용자가 이미 익숙한 크기 없음.

## 4. PM 검증

✅ 마스터 플랜과 충돌 없음 (게임 품질 개선)
✅ 불변 원칙 위반 없음 (오프라인 유지, 인터랙션 패턴 변경 없음)
✅ 게임 규칙 변형 없음 (그리드 크기만 조정)
✅ GC 의견: master 크기 축소가 도전성 저하 우려 있으나, **유일해 미보장 게임은 도전 의미 자체가 사라짐** — 축소 수용.
