## 개선 1: 마지막 변경 강조 (Last-Change Highlight) — 공통

### 1.1 목적
- 직전 입력 셀을 0.4초간 펄스/하이라이트로 시각적 피드백 강화
- 사용자가 자신의 입력 위치를 즉시 인지 → 인지 부하 ↓
- 참조 사이트 13/13 게임 공통 기본 제공

### 1.2 기능 요구사항
- 셀 입력(값 변경) 직후 해당 셀에 펄스 효과 표시
- 펄스: 0.4초 동안 배경색이 노란빛 → 원래 색 페이드
- 적용 게임: 13게임 전체
- 비활성화 옵션: 설정 화면에서 ON/OFF (기본 ON)

### 1.3 비기능 요구사항
- 60fps 유지 (AnimationController 활용)
- 게임별 보드 위젯에 최소 침습적 변경
- 일시정지/포기 시 자동 종료

## 개선 2: 자동 완성 표시 (Auto-Complete Visual) — 게임별

### 2.1 목적
- 완성된 행/열/숫자/영역을 자동으로 dim 처리하여 미완성 영역에 집중 가능
- 풀이 속도 향상 + 실수 감소

### 2.2 게임별 적용 범위 (이번 사이클)

| 게임 | 적용 항목 |
|------|----------|
| **Sudoku/Killer/Jigsaw** | 숫자 패드에서 이미 9개 사용된 숫자 50% opacity |
| **Binairo** | 행/열 0/1 개수가 N/2에 도달하면 해당 행/열 옅은 회색 배경 |
| **Nonograms** | 행/열 완성 시 힌트 숫자 회색 + 자동 X 표시 옵션 |

### 2.3 비기능
- 설정에서 ON/OFF 가능 (기본 ON)
- 풀이 정답 여부와 무관 (단순 카운트 기반)

## 개선 3: 체크포인트 (Checkpoint) — 공통

### 3.1 목적
- 게임 중 1슬롯 스냅샷 저장 + 복원
- Try-and-error 전략을 안전하게 시도 가능
- 고난도 퍼즐 접근성 ↑

### 3.2 기능 요구사항
- 컨트롤 바에 체크포인트 버튼 추가 (📌 아이콘)
- **저장 모드**: 체크포인트가 없을 때 → 현재 상태 저장
- **복원 모드**: 체크포인트가 있을 때 → 저장된 상태로 복원
- 길게 누르기 → 체크포인트 삭제
- 적용 게임: 13게임 전체
- 일시정지에서도 사용 가능

### 3.3 데이터
- 메모리에만 저장 (앱 종료 시 사라짐)
- 상태 객체 전체 스냅샷 (board, mistakeCount, hintCount, elapsedSeconds, undoStack)

### 3.4 다국어 키
- `checkpoint.save`: 체크포인트 저장
- `checkpoint.restore`: 체크포인트 복원
- `checkpoint.saved`: 체크포인트가 저장되었습니다
- `checkpoint.restored`: 체크포인트로 복원되었습니다
- `checkpoint.cleared`: 체크포인트가 삭제되었습니다
