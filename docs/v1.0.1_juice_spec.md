## 2. W1 상세 사양

### W1-1: 셀 입력 스케일 애니메이션
- 트리거: 사용자가 셀에 값 입력 (notifier `_applyValue` 등)
- 효과: 셀이 1.0 → 1.15 → 1.0 스케일 (150ms, `Curves.easeOutBack`)
- 위치: 공통 위젯 `AnimatedCell` 신설, 게임별 보드 위젯에서 사용
- 저성능 대응: 설정에서 "효과 줄이기" ON 시 50ms로 단축

### W1-2: 행/열/박스 완성 펄스 (스도쿠/킬러/직소)
- 트리거: 셀 입력 후 행/열/3×3 박스가 1~9 모두 채워질 때
- 효과: 해당 셀들이 옅은 초록 배경(alpha 0.25)으로 200ms 펄스
- 코드 위치: `sudoku_board_widget.dart`, `killer_sudoku_board_widget.dart`, `jigsaw_sudoku_board_widget.dart`
- 사운드: W2-1의 "complete_line.wav" 연동

### W1-3: 같은 숫자 입력 시 보드 전체 옅은 펄스 (스도쿠계)
- 트리거: 숫자 입력 → 보드에 동일 숫자가 있는 셀들이 100ms간 강조
- 효과: 동일 숫자 셀 배경 alpha 0.15 추가
- 이미 정적 하이라이트 존재 → 동적 펄스 추가

### W1-4: 잘못된 입력 셀 shake (13게임)
- 트리거: `wasMistake == true` 시점
- 효과: 셀이 좌우로 ±4px 진동 (3회, 총 200ms)
- 햅틱: 기존 heavyImpact 유지
- 사운드: W2-1의 "mistake.wav" 연동

### W1-5: 보드 진행률 바 (13게임)
- 위치: 상태바 바로 아래 또는 통합
- 게임별 진행률 정의:
  - 스도쿠/킬러/직소: filled / (81 - initial) 비율
  - 비나이로/음양: filled / totalCells
  - 노노그램: 채움 / 정답 채움 수
  - 지뢰찾기: revealed / safeCells
  - 라이트업: lit cells / total whites
  - 등
- 시각: 높이 4px LinearProgressIndicator (Material 토큰)
- 애니메이션: 200ms

### W1-6: 결과 화면 통계 카운트업 (13게임)
- 트리거: 결과 화면 진입 시
- 효과: 시간/실수/힌트 숫자가 0 → 실제값으로 0.8초 카운트업
- 위치: 공통 `CountUpText` 위젯
- easing: `Curves.easeOutCubic`

### W1-7: 배지 팝업 스케일+회전+글로우 (13게임)
- 트리거: `lastNewBadges`에 새 배지가 있을 때
- 효과: 배지 칩이 스케일 0 → 1.2 → 1.0 + 회전 -10° → 0° + 글로우 (500ms)
- 사운드: W2-1의 "badge.wav" 연동
- 위치: 공통 `AnimatedTrophy` 위젯 (배지 칩 wrapper)

### W1-8: 숫자 패드 9개 완성 시 페이드 (숫자 입력 게임)
- 트리거: 보드에 특정 숫자가 9개 채워질 때 (스도쿠) / size개 (futoshiki/skyscrapers)
- 효과: 해당 숫자 패드 버튼이 0.3 opacity로 페이드 (300ms)
- 비활성화: tap 시 햅틱 없음 (이미 다 채움)
- 스도쿠는 이미 일부 구현 → 다른 5게임 확장

## 3. W2-1 사운드 시스템 상세 사양

### 3.1 패키지 선정
- **audioplayers** (^6.0.0) — Flutter 공식 권장, 라이센스 MIT
- 대안: `just_audio` 검토하되 audioplayers 우선

### 3.2 사운드 자산 (라이센스 안전)
**소스**: 
- **Mixkit Free** (https://mixkit.co/free-sound-effects/) — CC0/Mixkit License (상업 사용 무료, 출처 표기 불필요)
- **Freesound.org CC0** (https://freesound.org) — CC0 라이센스만 사용

**6종 효과음 (각 < 100KB, WAV/MP3)**:

| 키 | 트리거 | 사양 | 추천 소스 |
|----|--------|------|----------|
| `click` | 셀 입력 | 50ms, 부드러운 틱 | Mixkit "Light click" |
| `mistake` | 잘못된 입력 | 200ms, 가벼운 부저음 | Mixkit "Error 1" |
| `line_complete` | 행/열/박스 완성 | 300ms, 짧은 차임 | Mixkit "Bell chime" |
| `game_complete` | 게임 완성 | 1000ms, 팡파레 | Mixkit "Achievement bell" |
| `badge` | 새 배지 획득 | 500ms, 별 사운드 | Mixkit "Magic spell" |
| `hint` | 힌트 사용 | 400ms, 마법 반짝임 | Mixkit "Magic wand" |

전체 약 600KB. APK 크기 영향 미미.

### 3.3 SoundManager 클래스
```dart
class SoundManager {
  static final SoundManager _instance = SoundManager._();
  factory SoundManager() => _instance;
  SoundManager._();

  final Map<String, AudioPlayer> _players = {};
  bool _soundEnabled = true;
  
  Future<void> preload() async { ... }  // 앱 시작 시 호출
  
  Future<void> play(String key) async {
    if (!_soundEnabled) return;
    final player = _players[key];
    if (player == null) return;
    try {
      await player.stop();
      await player.resume();
    } catch (_) {
      // 사운드 재생 실패 무시 (게임 진행 방해 X)
    }
  }
  
  void setEnabled(bool enabled) {
    _soundEnabled = enabled;
  }
}
```

### 3.4 통합 위치
- 앱 시작 시 `main.dart` 또는 `app.dart`에서 `SoundManager().preload()`
- 13게임 notifier의 적절한 위치에서 `SoundManager().play('click')` 등
- 설정 화면에 "사운드" 토글 추가
- 기존 SharedPreferences `soundEnabled` 키 활용

### 3.5 설정 화면 토글
```dart
SettingsService에 추가:
- bool get soundEnabled (이미 존재할 수 있음)
- setSoundEnabled(bool)
```

설정 화면 위젯에 사운드 토글 추가:
- 라벨: "사운드 / Sound / サウンド / 声音"
- 기본값: ON
- 토글 변경 시 즉시 SoundManager에 반영

## 4. 비기능 요구사항

### 4.1 성능
- 모든 애니메이션 60fps 유지
- 저성능 폰 대응: 설정에 "효과 줄이기" 토글 (기본 OFF)
- 사운드 preload는 main에서 비동기

### 4.2 접근성
- 사운드/햅틱/효과 각각 독립 토글
- 시각 효과는 화려하지만 산만하지 않도록 alpha 제한

### 4.3 호환성
- audioplayers 6.0+ Android 5.0+ 지원 (우리는 minSdk 31이라 OK)
- 사운드 권한 불필요 (시스템 볼륨 사용)
