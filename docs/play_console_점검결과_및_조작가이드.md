## B. 직접 조작 ① — 등록정보 저장 (지금 입력돼 있음)

현재 **"기본 스토어 등록정보"** 페이지(다른 탭)에 아래 내용이 입력돼 있습니다. 검토 후 저장하세요.

- 앱 이름: `K-퍼즐 : 광고 없는 퍼즐 게임` (18/30)
- 간단한 설명: `광고 없이, 인터넷 없이. 스도쿠부터 카쿠로까지 13가지 논리 퍼즐 한 앱에.`
- 자세한 설명: K-퍼즐 13게임 본문 (타쿠주 반영 완료)

**조작**: 페이지 하단 우측
1. 내용 확인 → **[저장]** 클릭
2. (관리형 게시 OFF이므로) 저장 시 검토 대기열에 올라가고, 승인되면 자동 공개됩니다.

> 만약 입력이 사라졌다면 `docs/play_store_listing_K_Puzzles.md`의 텍스트를 복사해 다시 붙여넣으세요.

## C. 직접 조작 ② — 그래픽 교체 (같은 페이지 "그래픽" 섹션)

1. **앱 아이콘** → 기존 삭제 후 업로드:
   `store_content/k_puzzles_assets/app_icon_512_kpuzzles.png`
2. **그래픽 이미지(피처 그래픽)** → 기존 삭제 후 업로드:
   `store_content/k_puzzles_assets/feature_graphic_1024x500_kpuzzles.png`
3. (권장) **스크린샷** → 13게임 다양성 보강 (현재 스도쿠 위주). 다른 게임 화면 캡처 후 추가.
4. **[저장]** 클릭

## D. 직접 조작 ③ — 앱 콘텐츠 선언 재확인 (선택, 읽기용)

좌측 메뉴 **정책 → 앱 콘텐츠**(또는 대시보드 "앱 설정 더보기")에서 아래가 모두 ✅인지 확인:
- 개인정보처리방침 URL · 앱 액세스 권한 · 광고(없음) · 콘텐츠 등급 · 타겟층 · 데이터 보안(수집 없음) · 정부/뉴스/금융(해당 없음)

> 이름·게임수가 바뀌어도 데이터 수집·콘텐츠 성격은 동일하므로 보통 그대로 유효합니다.

## E. 직접 조작 ④ — 프로덕션 출시 (핵심)

좌측 **테스트 및 출시 → 프로덕션** 에서:
1. **[새 버전 만들기]** 클릭
2. **App Bundle 업로드** — 프로덕션용 새 AAB
   - ⚠️ alpha에 올라간 v1.0.0(코드 3)이 아니라, **이름 변경 + 13게임 반영된 최신 빌드**를 새 버전코드로 빌드해 업로드
   - 빌드 명령은 `CLAUDE.md`의 "AAB 빌드" 참고 (버전코드 상향 필수)
3. **출시 노트** 작성 (예: "앱 이름을 K-퍼즐로 변경, 퍼즐 13종으로 확장, 비나이로→타쿠주")
4. **[버전 검토]** → 오류/경고 확인
5. (권장) **단계적 출시** 비율 설정 (예: 20%)
6. **[검토를 위해 출시 시작 / Google에 전송]** 클릭 → 검토 후 게시

> 출시 시점을 직접 통제하려면, 먼저 **게시 개요 → 관리형 게시 사용 설정**을 켜두면 승인 후 수동으로 "게시" 버튼을 눌러 공개할 수 있습니다.

## F. 권장 순서 요약

1. 등록정보 텍스트 **저장** (B)
2. 아이콘·피처 그래픽 **교체** (C)
3. (선택) 앱 콘텐츠 선언 **확인** (D)
4. 최신 AAB **빌드** → 프로덕션 **새 버전 생성·제출** (E)
5. 검토 통과 → 공개
