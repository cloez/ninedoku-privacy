## 2. 항목별 상세 설계

### H1. L2 핵심 셀 초록 테두리

#### 목적
Naked Pair, Hidden Pair, Pointing Pair 등 **다중 셀 기법**에서 "어디 두/세 셀이 핵심인지"를 시각으로 즉시 인지하도록 한다. 중급자 만족도의 결정 포인트.

#### 시각 명세
- **대상**: L2 진입 시점에 보드에 렌더.
- **색상**: `#10B981` (AppColors.successGreen, 기존 토큰 재사용)
- **두께**: 2.5dp 외곽 테두리 (`Border.all`)
- **셀 내부**: 변화 없음 (배경/숫자 그대로). 펄스 없음 (L1과 구분).
- **지속**: L2 단계가 종료될 때까지 정적 유지.
- **Single 기법 단일 셀일 경우**: 기존 `hintTargetCell` 표시와 중복 → **테두리만 표시, 다른 강조 미적용**.

#### 다국어 — 신규 키 없음
시각 효과만 추가. 텍스트는 기존 `hint.l2.*` 재사용.

#### 상태 변경
`HintResult`에 신규 필드 추가:
```dart
final List<({int row, int col})> keyCells;  // L2 강조 대상 셀
```
- `naked_single`/`hidden_single`: `[targetCell]` 1개
- `naked_pair`/`hidden_pair`: 페어 셀 2개
- `naked_triple`: 트리플 셀 3개
- `pointing_pair`/`box_line_reduction`: 같은 라인의 핵심 셀 2~3개
- `x_wing`: 4개 코너 셀

#### 모션 감소 대응
정적 테두리이므로 별도 처리 불필요. L1 펄스만 모션 감소 시 정적 색상 전환.

#### 접근성
- 색상에만 의존하지 않도록 **L2 메시지 본문에 좌표 명시**: "박스 5의 (5,4), (5,6) 칸을 살펴보세요" (다중 셀 기법에 한함).
- 신규 키: `hint.l2.coords` (4언어) — 좌표 접미사. 예: `" — ({r},{c}), ({r},{c})"`.

### H2. L3 후보 숫자 자동 메모 시각화 + 소거 X (★ 본 사이클 핵심)

#### 목적
"왜 그 답인지"를 **시각으로 증명**한다. 챔피언 v2 D.1.2에서 "입문자 학습 가치의 70%"라 평가한 결정타.

#### 시각 명세
L3 진입 시 자동 렌더:

1. **대상 셀 + 같은 영역(박스/행/열) 빈 셀들**의 후보 숫자를 셀 내부에 자동 표시.
2. **후보 폰트**: 사용자 메모 폰트보다 약간 작게 (사용자 메모 10sp → 힌트 메모 9sp), **색상 분리**:
   - 사용자 메모: 기존 `AppColors.textSecondary` 유지
   - 힌트 자동 메모(잔존 후보): `#6B7280` (회색 500, neutral)
   - **소거되는 후보**: `#EF4444` (red 500) + 가운데 사선 (대각선 strike-through)
   - **정답 후보**: `#10B981` (green 500) 굵게
3. **3x3 그리드 배치**: 1~9 각각 고정 위치 (사용자 메모와 동일 그리드). 미해당 숫자는 공백.
4. **L4로 전환 시 자동 메모 사라짐** (정답만 채워짐).
5. **사용자가 셀 탭/입력 시 즉시 사라짐** (게임 진행 우선).

#### 다국어 — 신규 키 없음 (시각 효과 위주)
기존 `hint.l3.*` 메시지 그대로 유지. 단, 메시지 첫 줄을 1줄 단축 권고:
- 현재 예: "같은 영역의 다른 칸들에는 숫자 {n}이(가) 들어갈 수 없어요. 그래서 이 칸만 가능해요"
- 권고: 유지하되 텍스트와 시각이 **상호 보강**. 텍스트 수정은 v2.1로 이월.

#### 상태 변경
`HintResult`에 신규/기존 필드 활용:
```dart
final Map<({int row, int col}), Set<int>> autoMemo;   // 셀별 잔존 후보
final Map<({int row, int col}), Set<int>> eliminated; // 셀별 소거된 후보
final ({int row, int col})? answerCell;               // 정답 셀
final int? answerValue;                               // 정답 숫자
```

엔진(`hint_engine.dart`)에서 L3 진입 시점에 채우고, 보드 위젯이 구독.

#### 모션 감소 대응
페이드 인 250ms → 0ms 즉시 전환.

#### 접근성
- 후보 메모는 시각 정보. **시각장애 대응**으로 L3 메시지 본문에 잔존/소거 후보를 텍스트로도 명시:
  - 신규 키 `hint.l3.candidates`: `"잔존 후보: {rest} · 소거: {strike}"` (4언어)
- TalkBack 스크린리더는 메시지를 읽음 (메모는 화면 그래픽이라 별도 시맨틱 라벨 추가하지 않음 — 인지 부하 회피).

### H3. L4 대상 셀 글로우 + scale 애니메이션

#### 목적
"갑자기 숫자가 박힌다"는 무미건조함을 해소하고 정답 공개의 **만족감**을 부여한다.

#### 시각 명세
L4 진입(자동 입력) 순간:

1. **글로우**: 대상 셀에 `BoxShadow(color: AppColors.successGreen.withOpacity(0.6), blurRadius: 16, spreadRadius: 2)` — 800ms 후 페이드아웃.
2. **숫자 scale**: `TweenAnimationBuilder<double>` 0.5 → 1.2 → 1.0 (Curves.easeOutBack, 400ms).
3. **셀 배경**: 일시적으로 `successGreen.withOpacity(0.18)` (300ms 후 원복).
4. **사운드/햅틱**: 기존 정답 입력 사운드와 동일 채널 — **중복 재생 방지** (이미 `inputNumber` 경로로 흐름).

#### 다국어 / 상태 변경 / 접근성
- 다국어: 변경 없음.
- 상태: `GameState`에 `lastL4HighlightUntil: DateTime?` (선택). 또는 위젯 내부 상태로 처리 가능 — DEV 판단에 위임.
- 모션 감소: scale/글로우 모두 생략, 배경 색상만 300ms 표시.
- 접근성: 시각 효과만 추가. 텍스트 안내는 기존 `hint.answer.*` 유지.

### H4. 클리어 화면 "사용 기법 기록"

#### 목적
"이번 게임에서 자동 감지된 기법" 노출로 **학습 동기** 부여. 차기 사이클의 기법 배지 인프라가 됨.

#### 시각/텍스트 명세
- 위치: 게임 클리어 모달(또는 결과 화면) 하단, 시간/실수/힌트 카운트 다음.
- 표시 양식:
  - `이번 게임에서 학습한 기법: Hidden Single, Naked Pair`
  - 0개일 경우 섹션 자체 비표시.
- 클릭 시: 해당 기법 사전 모달 열림 (H5와 연동).

#### 다국어 — 신규 키
| 키 | 한 | 영 | 일 | 중 |
|---|---|---|---|---|
| `result.usedTechniques.title` | 이번 게임에서 학습한 기법 | Techniques learned | 学んだテクニック | 学到的技巧 |
| `result.usedTechniques.empty` | (해당 없음 — 표시 생략) | — | — | — |

기법 이름은 기존 `hint.technique.{key}` 키 재사용 → **신규 키 1개만 필요**.

#### 상태 변경
`GameState`(또는 `SudokuGameState`)에 추가:
```dart
final Set<SolvingTechnique> usedTechniques;
```
- L2 진입 시점에 해당 기법을 Set에 추가 (L1 단계에는 기법이 확정되지 않을 수 있으므로 L2 시점 채택).
- 게임 새로 시작 시 초기화.
- 영속화 대상 아님 (현재 진행 중 세션만).

#### 모션 감소 / 접근성
- 변경 없음. 정적 텍스트.
- 시맨틱 라벨: 기본 `Text` 위젯이 자동 처리.

### P1. 비용 정책 — L1 무료화 합의 안건

#### 옵션 비교

| 옵션 | L1 | L2 | L3 | L4 | 사이클 총합 | 사용자 인식 |
|---|---|---|---|---|---|---|
| **(A) 현행 유지** | +1 | 0 | 0 | +1 | **+2** | "영역만 봐도 카운트 든다" |
| **(B) L1 무료화** ★ | 0 | +1 | 0 | +1 | **+2** | "영역 보고 스스로 풀면 무료" |
| (C) L1+L4 무료, L2/L3 비용 | 0 | +1 | +1 | 0 | +2 | "정답은 무료, 학습은 유료" — 직관 위배 |

#### GD 의견 / 추천: **(B) L1 무료화**
- **근거 1**: 챔피언 v2 D.2.2 권고 직접 채택. 입문자가 "어디?"만 확인하고 스스로 푸는 동선을 칭찬받음.
- **근거 2**: 총합 +2 유지 — 등급 임계값/통계/실적 영향 0.
- **근거 3**: 시장 조사 — Sudoku.com/Andoku 모두 "정답 공개 1회"에만 비용. L1 영역 안내를 무료화하는 것이 학습 친화선.
- **근거 4**: 옵션 (C)는 "정답 공개를 무료화"하여 학습 효과 정반대 — 반려.

#### 합의 필요
CLAUDE.md 원칙상 단독 결정 금지. **GD + GC + 사용자 합의** 후 적용. 합의 전에는 코드 변경 보류.
