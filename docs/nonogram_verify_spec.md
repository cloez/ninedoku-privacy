## 1. 변경 배경

현재 노노그램은 스도쿠 스타일의 즉시 실수 검증 방식을 사용:
- 채움(1)을 정답이 아닌 위치에 놓으면 실수 +1
- X(0) 표시는 검증 대상이 아님 (비대칭)
- 풀이 도중 "여기 채움이 정답이 아니다"가 즉시 노출 → 추리 즐거움 반감

정통 노노그램 (puzzle-nonograms.com 등)은:
- X와 채움을 자유롭게 표시 (둘 다 풀이 도구)
- 실수 카운트 없음
- "확인" 버튼으로 명시적 검증
- 사용자가 풀이 자신감이 들 때 검증 요청

## 2. 변경 사항

### 2.1 실수 즉시 검증 제거
- `_applyValue`에서 `mistakeCount++` 로직 제거
- 채움/X/지우개 모두 항상 허용
- 실수 햅틱(heavyImpact) 제거 → 입력 selectionClick만 유지

### 2.2 자동 완료 판정 유지 + 확인 버튼 추가
**자동 완료**: 보드가 정답과 일치하면 자동으로 완료 (현재 유지)
- 사용자가 정답대로 모두 채우면 자동 완료
- "확인" 버튼을 누르지 않아도 OK

**확인 버튼**: 사용자가 명시적으로 검증 요청
- 컨트롤 바에 "확인(✓)" 버튼 추가
- 탭 시:
  - 정답이면 → 즉시 완료 (자동 완료와 동일)
  - 정답 아니면 → 토스트 "아직 풀이가 완성되지 않았어요" + 약한 햅틱
- **틀린 위치를 알려주지 않음** (정통 노노그램 스타일)

### 2.3 결과 화면 — 실수 → 다른 지표
**기존**:
- 시간, 난이도, 실수, 힌트

**변경**:
- 시간, 난이도, 채움 셀 수, 힌트
- 또는 단순히 "실수" 항목 제거 (4개 → 3개)

### 2.4 등급 산정 변경
**기존**: 실수 + 힌트 + 시간 기반
**변경**: 힌트 + 시간만 사용 (실수 부분 제거)

`NonogramGrade.evaluate`에서 `mistakes` 파라미터 사용 안 함.

### 2.5 배지 영향
실수 관련 배지가 있다면 시간/힌트/완료 횟수 기반으로 대체.
`nono_perfect`(PerfectClear 조건) — 기존 `record.grade == 'S'` 유지. S 등급은 노힌트 + 기준시간 이내로 재정의.

## 3. UI 변경

### 3.1 상태 바
- 시간, 실수, 힌트 → **시간, 힌트** (실수 표시 제거)
- 또는 실수 자리에 "남은 채움 / 전체 채움" 같은 진행률 표시 (선택)

### 3.2 컨트롤 바
- 기존: [채우기] [지우기] [↩되돌리기] [💡힌트] [📌체크포인트]
- 변경: [채우기] [✕크로스] [↩되돌리기] [💡힌트] [📌체크포인트] [✓확인]
- "확인" 버튼은 항상 활성, 탭 시 완료 판정

### 3.3 햅틱
- 입력: selectionClick (유지)
- 실수: heavyImpact (제거)
- 완료: heavyImpact (유지)
- 확인 실패: 약한 진동(selectionClick × 2 또는 mediumImpact)

## 4. 다국어 키 추가

| 키 | 한국어 | 영어 | 일본어 | 중국어 |
|----|--------|------|--------|--------|
| `nonogram.verify` | 확인 | Check | 確認 | 检查 |
| `nonogram.verify.fail` | 아직 풀이가 완성되지 않았어요 | The puzzle isn't complete yet | まだパズルが完成していません | 谜题尚未完成 |

## 5. 영향 범위

### 변경 파일
- `lib/games/nonograms/nonogram_notifier.dart` — mistakeCount 제거 + 확인 메서드 추가
- `lib/games/nonograms/nonogram_state.dart` — Grade.evaluate에서 mistakes 파라미터 무시
- `lib/games/nonograms/screens/nonogram_game_screen.dart` — 컨트롤 바에 확인 버튼 추가, 상태 바에서 실수 제거, 결과 화면 통계 수정
- `lib/shared/l10n/app_strings.dart` — 4개 언어 키 추가

### 영향 받지 않음
- nonograms generator/solver (게임 로직 무관)
- 다른 12게임 (노노그램만 특수 처리)
- 저장된 게임 (mistakeCount 필드 유지하되 0으로 고정/표시 안 함)

### 백워드 호환
- 기존에 저장된 NonogramState의 `mistakeCount` 필드는 그대로 유지 (JSON 호환)
- UI에서 표시하지 않을 뿐, 데이터 모델 변경 없음
