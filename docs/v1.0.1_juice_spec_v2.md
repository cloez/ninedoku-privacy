## 🔴 블로커 해소

### B1: W1-5 진행률 정의 일관화
- **변경**: 모든 게임에서 "결정된 셀 / 결정 필요 셀" 통일 정의
- **지뢰찾기 스포일러 회피**: revealed 셀 수가 아닌 "남은 안전 셀 수" 표시 (사용자가 풀이 진행도 인지 가능, 정답 정보 노출 X)
- **사양**: 게임별 progress getter를 `XxxState`에 추가 (0.0~1.0 double)
- 노노그램: 정답 셀과 무관하게 "채움 + X 표시 / 전체" 비율

### B2: 모션 감소 시스템 연동
- **변경**: 공통 헬퍼 `motionScale(BuildContext)` 추가
  ```dart
  double motionScale(BuildContext context) {
    if (MediaQuery.disableAnimationsOf(context)) return 0.0;
    if (SettingsService(prefs).reduceEffects) return 0.3;
    return 1.0;
  }
  ```
- 모든 애니메이션 duration에 `motionScale` 곱:
  ```dart
  Duration(milliseconds: (150 * motionScale(context)).round())
  ```
- motionScale == 0이면 효과 스킵, 즉시 최종 상태

### B3: Mixkit 라이센스 정정
- **재기재**: Mixkit Free의 정확한 조건:
  - ✅ 상업/비상업 무료 사용
  - ✅ 출처 표기 권장이나 필수 아님
  - ❌ 재배포 불가 (앱에 임베드는 OK)
  - ❌ Mixkit 소속물로 판매 금지
- **추가 안전망**: Freesound.org CC0 라이센스 자산 우선 검토, Mixkit는 보완
- **법무 검증 후 확정**: 자산 다운로드 전 `docs/sound_assets_license.md`에 각 파일 출처/라이센스 명시 후 사용

### B4: 효과 동시 발사 우선순위 정의
- **셀 입력 시 효과 시퀀스** (병렬 발사, 충돌 없음):
  1. 시각 (스케일 애니, 0~150ms)
  2. 햅틱 (selectionClick, 0ms)
  3. 사운드 (click, 0ms)
- **잘못된 입력**:
  1. 시각 (shake, 0~200ms)
  2. 햅틱 (heavyImpact, 0ms)
  3. 사운드 (mistake, 0ms)
- **완성 시 (배지 + 카운트업 동시)**:
  1. 카운트업 통계 (0~800ms)
  2. 결과 화면 페이드인 완료 후 (1000ms 시점)
  3. 새 배지 있으면 배지 팝업 (스케일+회전+글로우, 500ms)
  4. 배지 팝업 시작 시 badge 사운드
- **시퀀스 보장**: `Future.delayed`로 명시적 순서

### B5: shake 진폭 셀 크기 비례
- **변경**: 진폭을 셀 크기 비례로 정의
  ```dart
  final amplitude = cellSize * 0.08; // 5% of cellSize
  ```
- 노노그램 20×20 작은 셀에서도 인접 침범 안 됨
- 최소 진폭 2px, 최대 6px (clamp)

### B6: 카운트업 시퀀스 명시
- 결과 화면 진입 시점부터:
  - 0~300ms: 결과 화면 페이드인 (기본)
  - 300~1100ms: 통계 카운트업 (시간/실수/힌트)
  - 1100~1600ms: 새 배지 팝업 (있을 때)
- 카운트업 중 사용자가 화면 탭하면 즉시 최종값 표시 (skip)

### B7: 사운드 동시 재생 정책
- **AudioPlayer 초기화 옵션**:
  ```dart
  AudioPlayer(playerId: 'sfx_click')
    ..setPlayerMode(PlayerMode.lowLatency)
    ..setReleaseMode(ReleaseMode.stop);
  ```
- 각 효과음에 **전용 AudioPlayer 인스턴스** (충돌 방지)
- **Throttle**: 같은 키의 연속 재생은 최소 50ms 간격 (사용자가 빠르게 셀 입력 시 click 연속 발사 안 됨)
- 충돌 방지: 새 재생 전 `stop()` 호출

## 🟡 권장 수정 반영

### W1-2 셀 너무 작은 폰 대응
- 펄스 alpha를 셀 크기 비례:
  - cellSize > 40dp: alpha 0.25
  - cellSize ≤ 40dp: alpha 0.35 (가시성 강화)
  
### 일시정지/재개 사운드 추가
- 7번째 사운드 키 `pause` 추가 (300ms, 부드러운 톤)
- 게임 일시정지 진입 시 재생

### 새 게임 시작 사운드 (P2 백로그로 이관)
- 너무 빈번 → 사운드 피로 우려, 일단 제외

### 카운트업에 미세 틱 사운드
- 100ms 간격으로 짧은 click (volume 0.3)
- 통계 카운트업 중 (시간만, 실수/힌트는 제외)
- 설정의 "사운드" 토글에 종속

## 추가 답변 (UX 질문)

1. **시즈 모드(저전력)에서 사운드 OFF?** — Android Battery Saver 감지하여 자동 OFF (`SystemChannels`)
2. **사운드 파일 포맷?** — OGG Vorbis 우선 (압축률 높음), 폴백 MP3
3. **첫 재생 지연 대비?** — 앱 시작 시 `preload()` 완료까지 대기 (스플래시 화면 또는 백그라운드)
4. **노노그램 확인 버튼 사운드?** — 성공: line_complete 재활용 / 실패: mistake 재활용
5. **체크포인트 사운드?** — 별도 사운드 없음 (toast 알림만, 빈번 발생 우려)
6. **백그라운드 → 포그라운드 복귀 사운드 정지?** — `didChangeAppLifecycleState`에서 `_player.stop()`
7. **국가별 사운드 차이?** — 불필요 (게임 효과음은 글로벌 공통)
