# K-퍼즐 메인 화면 디자인 가이드

## 1. 문서 개요

본 문서는 13개의 퍼즐게임을 제공하는 모바일 앱 **K-퍼즐**의 메인 화면 디자인 방향과 구성 요소를 설명한다.

기존 화면은 정보 전달에는 문제가 없지만, 카드와 배경의 색상 변화가 적고 게임별 시각적 차이가 약해 전체적으로 정적인 인상을 준다.  
개선 디자인은 **프리미엄 캐주얼 퍼즐게임 앱**을 목표로 하며, 밝고 생동감 있는 색상, 통일된 게임 아이콘, 진행 현황 강조, 카드별 시각적 차별화를 적용한다.

---

## 2. 디자인 목표

### 2.1 핵심 목표

- 첫 화면에서 다양한 퍼즐을 탐색하는 즐거움 제공
- 13개 게임을 한눈에 구분할 수 있는 시각적 구조 구현
- 오늘의 퍼즐과 연속 플레이 기록을 강조하여 재방문 유도
- 밝고 친근하지만 유아용처럼 보이지 않는 프리미엄 디자인 유지
- 새로운 퍼즐이 추가되어도 자연스럽게 확장 가능한 구조 설계

### 2.2 주요 디자인 키워드

- Exciting
- Premium Casual
- Colorful
- Friendly
- Gamification
- Clean
- Expandable

---

## 3. 전체 화면 구조

메인 화면은 다음의 네 가지 영역으로 구성한다.

1. 상단 앱 바
2. 오늘의 진행 현황 영역
3. 퍼즐게임 카드 목록
4. 하단 내비게이션

화면은 세로 스크롤 방식으로 구성하며, 퍼즐게임 카드는 기본적으로 2열 그리드를 사용한다.

---

## 4. 상단 앱 바

### 4.1 구성

- 중앙: 앱 이름 `K-퍼즐`
- 우측: 설정 아이콘
- 좌측: 기본적으로 비워 두되, 추후 프로필이나 알림 기능 확장 가능

### 4.2 디자인

- 앱 이름은 굵고 명확한 한글 타이포그래피 사용
- 기본 색상은 짙은 인디고 또는 네이비 계열
- 설정 아이콘은 흰색 원형 버튼 안에 배치
- 버튼에는 약한 그림자와 눌림 효과 적용
- 상단 여백을 충분히 확보하여 답답하지 않게 구성

### 4.3 권장 스타일

- 제목 크기: 24~28sp
- 제목 굵기: Bold 또는 ExtraBold
- 설정 버튼 크기: 40~44dp
- 설정 아이콘 크기: 22~24dp

---

## 5. 오늘의 진행 현황 영역

### 5.1 목적

사용자가 앱을 실행하자마자 오늘의 플레이 상태와 연속 기록을 확인할 수 있도록 한다.

### 5.2 표시 정보

- 오늘의 퍼즐 진행 수
- 전체 퍼즐 수
- 연속 플레이 일수
- 진행 상태 아이콘
- 연속 기록 아이콘

예시:

- 오늘의 퍼즐 `0/13`
- 연속 `0일`

### 5.3 디자인 방향

- 하나의 넓은 히어로 카드 형태로 구성
- 인디고 또는 바이올렛 계열의 강한 그라데이션 사용
- 좌우 두 영역으로 구분
- 중앙에 얇은 구분선 배치
- 오늘의 퍼즐에는 달력 또는 별 아이콘 사용
- 연속 기록에는 불꽃 아이콘 사용
- 숫자는 일반 텍스트보다 크게 표시
- 배경에 작은 별, 퍼즐 조각, 반짝임 등의 장식 요소 추가

### 5.4 게이미피케이션 요소

- 진행률이 증가하면 숫자와 아이콘에 애니메이션 적용
- 연속 기록이 일정 기준을 넘으면 불꽃 크기나 색상 변화
- 모든 퍼즐 완료 시 축하 효과 표시
- 완료 상태에서는 카드 배경에 빛나는 테두리 추가 가능

## 6. 퍼즐게임 카드

## 6.1 기본 구조

각 게임 카드는 다음 정보를 포함한다.

- 게임 번호
- 대표 아이콘
- 게임 이름
- 한 줄 설명
- 필요 시 상태 배지

기본 배치는 2열 카드형 그리드로 한다.

### 카드 예시 구조

```text
[번호]

[게임 아이콘]

게임 이름
게임 설명
```

또는 가로형 카드에서는 다음 구조를 사용할 수 있다.

```text
[번호] [게임 아이콘] [게임 이름]
                     [게임 설명]
```

### 6.2 카드 디자인

- 라운드 처리된 직사각형 카드
- 카드별로 서로 다른 파스텔 색상 또는 그라데이션 적용
- 테두리는 최소화하고 그림자로 깊이감 표현
- 아이콘 주변에는 은은한 광택이나 반사 효과 적용
- 카드 배경에는 작은 곡선, 반짝임, 퍼즐 패턴 등을 사용
- 텍스트 가독성을 위해 아이콘 영역과 텍스트 영역을 명확히 구분

### 6.3 카드 권장 규격

- 카드 모서리: 18~24dp
- 카드 간 간격: 12~16dp
- 화면 좌우 여백: 16~20dp
- 카드 내부 여백: 14~18dp
- 그림자 높이: 2~6dp
- 카드 최소 높이: 120~150dp

### 6.4 상호작용

- 터치 시 카드가 살짝 축소되는 효과
- 손을 떼면 원래 크기로 복귀
- 카드 선택 시 대표 색상이 잠시 강조
- 완료한 퍼즐은 체크 표시 또는 별 표시
- 새로 추가된 퍼즐은 `NEW` 배지 표시
- 잠금 상태가 있다면 자물쇠 아이콘과 해제 조건 표시

## 7. 게임별 시각 아이덴티티

각 게임은 한눈에 구분될 수 있도록 고유 색상과 아이콘을 사용한다.

| 번호 | 게임 | 대표 아이콘 | 권장 색상 |
|---|---|---|---|
| 01 | 스도쿠 | 숫자 격자와 연필 | 블루 |
| 02 | 비나이로 | 흑백 구슬 또는 이진 원형 | 민트 |
| 03 | 지뢰찾기 | 폭탄 | 바이올렛 |
| 04 | 음양 | 음양 구슬 | 핑크·퍼플 |
| 05 | 노노그램 | 픽셀 격자와 연필 | 오렌지 |
| 06 | 킬러 스도쿠 | 케이지가 표시된 숫자 격자 | 스카이블루 |
| 07 | 스타 배틀 | 빛나는 별 | 골드 |
| 08 | 라이트업 | 빛나는 전구 | 시안 |
| 09 | 후토시키 | 저울 또는 부등호 | 틸 |
| 10 | 텐트 | 숲과 텐트 | 코랄·오렌지 |
| 11 | 직소 스도쿠 | 퍼즐 조각 | 그린 |
| 12 | 빌딩 | 도시 건물 | 네이비·블루 |

아이콘 스타일은 서로 달라도 표현 방식은 통일해야 한다.

- 동일한 광원 방향
- 동일한 그림자 강도
- 동일한 3D 또는 반입체 표현
- 동일한 윤곽선 굵기
- 동일한 시각적 크기

## 8. 카쿠로 신규 게임 강조

카쿠로는 새롭게 추가된 퍼즐이므로 다른 카드보다 조금 더 강조한다.

### 적용 방법

- 카드 우측 상단에 `NEW` 배지 표시
- 핑크 또는 레드 계열 강조색 사용
- 첫 노출 시 작은 반짝임 애니메이션 적용
- 필요 시 2열 폭 전체를 사용하는 가로형 카드로 구성
- 신규 퍼즐 추가 시 동일한 구조를 재사용할 수 있도록 설계

### NEW 배지 권장 스타일

- 배경: 레드
- 글자: 흰색
- 모서리: 8~12dp
- 크기: 12~14sp
- 과도한 깜빡임은 사용하지 않음

## 9. 하단 내비게이션

### 9.1 메뉴

- 통계
- 배지
- 설정

### 9.2 디자인

- 둥근 모서리의 플로팅 내비게이션 바 형태
- 흰색 또는 아주 밝은 배경
- 상단에 약한 그림자 사용
- 선택된 메뉴는 아이콘과 텍스트를 강조색으로 표시
- 선택되지 않은 메뉴는 회색 또는 낮은 채도로 표시

### 9.3 권장 아이콘

- 통계: 막대그래프
- 배지: 트로피
- 설정: 톱니바퀴

### 9.4 선택 상태

- 아이콘 색상 변경
- 텍스트 굵기 증가
- 아이콘 아래 작은 점 또는 배경 캡슐 추가 가능

## 12. 배경 디자인

기존 단색 배경 대신 아주 약한 분위기 요소를 추가한다.

### 적용 요소

- 밝은 아이보리 또는 라벤더 그라데이션
- 반투명 원형 도형
- 매우 연한 퍼즐 조각 패턴
- 작은 별과 반짝임
- 화면 상단의 부드러운 빛 번짐

배경 장식은 카드보다 눈에 띄지 않아야 하며, 콘텐츠 가독성을 방해하지 않아야 한다.

## 13. 애니메이션과 모션

### 13.1 화면 진입

- 상단 진행 카드가 부드럽게 나타남
- 게임 카드는 위에서 아래로 순차적으로 페이드인
- 각 카드 사이에 약 30~60ms 정도의 간격 적용

### 13.2 카드 터치

- 0.96~0.98 배율로 축소
- 그림자가 잠시 약해짐
- 터치 해제 후 원래 크기로 복원

### 13.3 진행 상태

- 오늘의 퍼즐 숫자 증가 시 카운트업 애니메이션
- 연속 기록 갱신 시 불꽃이 짧게 커지는 효과
- 퍼즐 완료 시 별 또는 체크 애니메이션
- 전체 완료 시 작은 축하 파티클 효과

### 13.4 모션 원칙

- 짧고 명확하게 적용
- 일반 전환: 150~250ms
- 강조 전환: 300~500ms
- 반복 애니메이션은 최소화
- 저사양 기기에서도 부드럽게 작동하도록 구현

## 14. 반응형과 확장성

퍼즐 수가 증가할 가능성이 있으므로 고정된 13개 전용 구조로 만들지 않는다.

### 설계 원칙

- 게임 목록은 데이터 기반으로 렌더링
- 게임 번호는 자동 생성
- 게임별 색상, 아이콘, 설명, 상태를 데이터로 관리
- `NEW`, `잠금`, `완료`, `추천` 등의 상태 배지 지원
- 게임 수가 증가하면 스크롤로 자연스럽게 확장
- 태블릿에서는 3열 또는 4열 그리드 적용 가능

### 예시 데이터 구조

```json
{
  "id": "kakuro",
  "order": 13,
  "title": "카쿠로",
  "description": "합계 힌트로 숫자를 채우는 크로스워드 논리 퍼즐",
  "icon": "kakuro",
  "theme": "pink",
  "isNew": true,
  "isLocked": false
}
```

## 15. 접근성

- 텍스트와 배경의 대비 확보
- 색상만으로 상태를 구분하지 않음
- 아이콘과 텍스트를 함께 제공
- 최소 터치 영역 44×44dp 확보
- 폰트 크기 확대 시 레이아웃이 깨지지 않도록 설계
- 스크린리더용 접근성 라벨 제공
- 애니메이션 감소 설정 지원 고려

예시 접근성 라벨:

```text
스도쿠 게임 열기
오늘의 퍼즐 진행률 13개 중 0개
연속 플레이 기록 0일
```

## 16. 기존 화면 대비 주요 개선점

| 구분 | 기존 화면 | 개선 화면 |
|---|---|---|

| 카드 표현 | 동일한 배경과 유사한 구성 | 게임별 색상과 아이콘 차별화 |
| 진행 현황 | 단순 정보 표시 | 강한 히어로 카드로 강조 |
| 게임 아이콘 | 이모지 중심 | 통일된 일러스트 아이콘 |
| 게이미피케이션 | 제한적 | 연속 기록, 진행률, 완료 효과 강화 |

| 확장성 | 13개 중심 | 게임 추가를 고려한 데이터 구조 |

## 17. 개발 적용 우선순위

### 1단계: 핵심 레이아웃 개선

- 상단 진행 현황 카드
- 게임별 카드 색상 구분
- 게임 아이콘 스타일 통일
- 카드 여백과 타이포그래피 개선
- 하단 내비게이션 개선

### 2단계: 게이미피케이션 적용

- 오늘의 퍼즐 진행 애니메이션
- 연속 기록 강조

## 18. 최종 디자인 방향 요약

K-퍼즐 메인 화면은 단순한 게임 목록이 아니라, 사용자가 매일 새로운 퍼즐에 도전하고 기록을 쌓는 **퍼즐게임 허브**처럼 보여야 한다.

핵심은 다음과 같다.

- 강한 브랜드 컬러를 사용하는 상단 진행 카드
- 게임마다 다른 색상과 통일된 고품질 아이콘
- 직관적인 2열 카드 구조
- 오늘의 퍼즐과 연속 기록을 활용한 재방문 유도
- 신규 게임과 완료 상태를 명확히 보여 주는 배지 시스템
- 부드러운 애니메이션과 반응형 터치 효과
- 게임 수 증가를 고려한 확장 가능한 데이터 기반 설계

전체 화면은 화려하지만 복잡하지 않아야 하며, 사용자가 앱을 실행했을 때 바로 원하는 퍼즐을 찾고 즐겁게 선택할 수 있어야 한다.
