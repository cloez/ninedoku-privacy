# K-퍼즐 개별 게임 메인화면 디자인 가이드

## 1. 문서 목적

본 문서는 K-퍼즐 게임 허브의 시각 디자인 패턴을 개별 퍼즐게임의 메인화면에도 일관되게 적용하기 위한 디자인 가이드이다.

스도쿠를 대표 샘플로 사용하되, 다음 퍼즐게임에도 동일한 화면 구조와 컴포넌트 체계를 재사용할 수 있도록 설계한다.

- 스도쿠
- 비나이로
- 지뢰찾기
- 음양
- 노노그램
- 킬러 스도쿠
- 스타 배틀
- 라이트업
- 후토시키
- 텐트
- 직소 스도쿠
- 빌딩
- 카쿠로
- 향후 추가되는 퍼즐게임

핵심 목표는 개별 게임마다 고유한 분위기를 유지하면서도, 사용자가 어느 게임에 들어가더라도 같은 앱 안에 있다는 일관성을 느끼게 하는 것이다.

---

## 2. 디자인 목표

### 2.1 핵심 목표

- 게임 허브와 개별 게임 화면의 시각적 일관성 유지
- 각 게임의 개성을 색상, 아이콘, 배경 장식으로 표현
- 주요 행동인 `새 게임`과 `오늘의 퍼즐`을 즉시 인지 가능하게 배치
- 정보가 많아도 복잡해 보이지 않는 계층 구조 제공
- 신규 게임이 추가되어도 동일한 템플릿으로 빠르게 확장 가능하도록 설계
- 모바일 환경에서 한 손 조작과 터치 편의성 확보

### 2.2 디자인 키워드

- Premium Casual
- Playful
- Colorful
- Clear
- Consistent
- Reusable
- Accessible
- Expandable

## 3. 전체 화면 구조

개별 게임 메인화면은 다음의 5개 영역으로 구성한다.

1. 상단 앱 바
2. 게임 히어로 영역
3. 주요 액션 버튼 영역
4. 보조 메뉴 영역
5. 게임 소개 및 규칙 카드

기본 화면은 세로 스크롤 방식으로 구성한다.

```text
[상단 앱 바]

[게임 히어로 카드]

[새 게임]

[오늘의 퍼즐]

[통계] [배지]

[게임 소개 및 규칙]
```

## 4. 상단 앱 바

## 4.1 구성

- 좌측: 게임 허브 이동 버튼
- 중앙: 현재 게임 이름
- 우측: 도움말 버튼
- 우측 끝: 설정 버튼

### 스도쿠 예시

- 좌측 아이콘: 3×3 그리드
- 중앙 제목: `스도쿠`
- 우측 아이콘: `?`
- 우측 끝 아이콘: 톱니바퀴

## 4.2 디자인 규칙

- 배경은 투명 또는 매우 밝은 배경 사용
- 제목은 짙은 인디고 또는 네이비 색상
- 모든 아이콘 버튼은 동일한 크기와 형태 사용
- 버튼은 둥근 사각형 형태로 통일
- 약한 그림자 또는 소프트 글로우 적용
- 상태바와 충분한 상단 여백 확보

## 4.3 권장 규격

| 항목 | 권장값 |
|---|---:|
| 앱 바 높이 | 56~64dp |
| 제목 크기 | 24~30sp |
| 제목 굵기 | Bold 또는 ExtraBold |
| 아이콘 버튼 | 44~48dp |
| 아이콘 크기 | 22~26dp |
| 좌우 여백 | 16~20dp |

## 4.4 동작

- 허브 버튼: 게임 허브 메인화면으로 이동
- 도움말 버튼: 게임 방법, 조작법, 힌트 설명 표시
- 설정 버튼: 해당 게임 또는 앱 전체 설정 표시

## 5. 게임 히어로 영역

## 5.1 목적

현재 선택된 게임의 정체성과 분위기를 가장 강하게 전달하는 대표 영역이다.

## 5.2 구성 요소

- 게임 대표 아이콘 또는 일러스트
- 게임을 상징하는 색상
- 짧은 슬로건
- 작은 장식 요소
- 필요 시 진행 상태 또는 시즌 테마

### 스도쿠 예시

- 대표 아이콘: 숫자가 일부 채워진 3×3 또는 9×9 격자
- 슬로건: `마음을 편안하게, 한 칸씩`
- 보조 장식: 별, 퍼즐 조각, 반짝임
- 대표 색상: 블루 + 인디고 + 바이올렛

## 5.3 디자인 규칙

- 화면 너비 대부분을 사용하는 대형 라운드 카드
- 강한 그라데이션 또는 고유 배경 패턴 적용
- 대표 아이콘은 카드 중앙 또는 중앙 상단에 배치
- 슬로건은 하단 중앙에 배치
- 장식 요소는 과하지 않게 사용
- 카드와 배경 간 충분한 명도 차이 확보

## 5.4 권장 규격

| 항목 | 권장값 |
|---|---:|
| 카드 높이 | 180~240dp |
| 카드 모서리 | 24~32dp |
| 카드 내부 여백 | 20~28dp |
| 대표 아이콘 크기 | 88~128dp |
| 슬로건 크기 | 17~22sp |
| 카드 그림자 | 4~10dp |

## 5.5 게임별 적용 원칙

히어로 카드의 구조는 유지하고 다음 요소만 게임별로 변경한다.

- 대표 색상
- 대표 아이콘
- 슬로건
- 장식 요소
- 배경 패턴

## 6. 주요 액션 버튼

## 6.1 새 게임 버튼

개별 게임 메인화면에서 가장 중요한 CTA이다.

### 구성

- 플레이 아이콘
- 텍스트 `새 게임`
- 게임 대표 색상 기반 배경
- 버튼 상태별 시각 효과

### 디자인

- 가장 크고 강한 색상 사용
- 화면 너비 대부분을 차지하는 대형 버튼
- 둥근 캡슐 또는 라운드 직사각형
- 흰색 또는 고대비 텍스트 사용
- 터치 시 눌림 효과 적용

### 권장 규격

| 항목 | 권장값 |
|---|---:|
| 높이 | 64~76dp |
| 모서리 | 24~32dp |
| 텍스트 크기 | 20~26sp |
| 아이콘 크기 | 24~32dp |

### 동작

- 마지막 난이도 또는 기본 난이도로 바로 시작
- 게임 시작 전 난이도 선택이 필요한 경우 바텀시트 표시
- 진행 중인 게임이 있다면 `이어하기`를 우선 표시 가능

## 6.2 오늘의 퍼즐 버튼

### 목적

매일 재방문할 이유를 제공하는 핵심 게이미피케이션 기능이다.

### 구성

- 달력 아이콘
- 텍스트 `오늘의 퍼즐`
- 완료 여부
- 필요 시 연속 기록

### 디자인

- 새 게임 버튼보다 한 단계 낮은 강조
- 밝은 바이올렛, 라벤더, 민트 등 사용 가능
- 완료한 경우 체크 아이콘 또는 완료 배지 표시
- 연속 기록이 있으면 작은 불꽃 아이콘 추가 가능

### 상태 예시

```text
오늘의 퍼즐
오늘의 퍼즐 완료
오늘의 퍼즐 · 3일 연속
```

## 8. 게임 소개 및 규칙 카드

## 8.1 목적

처음 접하는 사용자가 게임의 개념과 기본 규칙을 빠르게 이해하도록 돕는다.

## 8.2 구성

1. 게임 소개 제목
2. 게임 소개 설명
3. 규칙 제목
4. 번호가 있는 규칙 목록
5. 시작 유도 문구

### 스도쿠 예시 제목

```text
스도쿠란?
```

### 스도쿠 소개 예시

```text
스도쿠는 간단한 규칙과 복잡한 해법을 지닌 논리 퍼즐입니다.
3×3 형태의 지역으로 분할된 9×9 격자 판에
1부터 9까지의 숫자를 채워야 합니다.
```

### 스도쿠 규칙 예시

1. 각 가로줄에는 1부터 9까지의 숫자가 오직 한 번씩만 들어가야 합니다.
2. 각 세로줄에는 1부터 9까지의 숫자가 오직 한 번씩만 들어가야 합니다.
3. 각 3×3 지역에는 1부터 9까지의 숫자가 오직 한 번씩만 들어가야 합니다.

### 시작 유도 문구

```text
첫 게임을 시작해 보세요!
```

## 8.3 디자인 규칙

- 밝은 흰색 또는 연한 라벤더 카드
- 게임 대표 색상 기반 테두리 또는 글로우 적용
- 제목 옆에 게임 관련 작은 아이콘 배치
- 규칙 번호는 컬러 원형 배지로 표시
- 규칙 사이에 옅은 구분선 사용
- 정보량이 많아도 읽기 쉽도록 충분한 행간 확보

| 본문 행간 | 1.4~1.6 |
| 규칙 번호 배지 | 30~36dp |

## 12. 배경 디자인

## 12.1 기본 배경

- 밝은 아이보리, 화이트 또는 라벤더 계열
- 화면 상단에 부드러운 색상 블롭 적용 가능
- 카드보다 채도가 낮아야 함

## 12.2 장식 요소

- 작은 별
- 퍼즐 조각
- 원형 버블
- 빛나는 점
- 얇은 곡선
- 게임별 상징 패턴

## 12.3 적용 원칙

- 콘텐츠보다 눈에 띄지 않게 사용
- 텍스트 주변에는 장식을 최소화
- 스크롤 시 장식이 반복되어 보이지 않도록 위치 다양화
- 저사양 기기에서는 정적 이미지로 대체 가능

- `새 게임` 버튼 강조
- 소개 카드 기본 펼침
- `첫 게임을 시작해 보세요!` 표시

## 13.2 진행 중인 게임이 있는 경우

새 게임보다 다음 구조를 우선할 수 있다.

```text
이어하기
새 게임
오늘의 퍼즐
```

이어하기 버튼에는 다음 정보를 표시할 수 있다.

- 난이도
- 진행률
- 경과 시간
- 마지막 플레이 시각

## 13.3 오늘의 퍼즐 완료

- 체크 아이콘 표시
- 버튼 색상을 완료 상태로 변경
- 완료 문구 표시
- 연속 기록 갱신 애니메이션 실행

## 13.4 잠금 또는 비활성 상태

- 투명도 감소
- 자물쇠 아이콘 표시
- 비활성 이유 제공
- 단순 회색 처리만 하지 않음

## 14. 모션 및 인터랙션

## 14.1 화면 진입

- 제목과 앱 바는 빠르게 표시
- 히어로 카드는 페이드인 또는 아래에서 등장
- 버튼은 30~60ms 간격으로 순차 등장
- 정보 카드는 마지막에 부드럽게 표시

## 14.2 버튼 터치

- 0.96~0.98 배율 축소
- 그림자 감소
- 짧은 햅틱 피드백
- 150~200ms 내 복원

## 14.3 오늘의 퍼즐 완료

- 체크 아이콘 팝업
- 작은 파티클 또는 별 효과
- 연속 기록 숫자 카운트업
- 전체 효과는 1초 이내 권장

## 14.4 접근성 설정

- 애니메이션 감소 옵션 지원
- 반복 애니메이션 최소화
- 깜빡임 효과 사용 금지
- 장식 애니메이션은 기능 이해에 필수적이지 않게 설계

## 17. 접근성

- 최소 터치 영역 44×44dp
- 텍스트와 배경의 충분한 대비
- 아이콘에 접근성 라벨 제공

## 22. 최종 디자인 원칙

K-퍼즐의 개별 게임 메인화면은 게임마다 완전히 다른 화면이 아니라, 동일한 디자인 시스템 안에서 게임별 테마만 달라지는 구조로 설계한다.

핵심 원칙은 다음과 같다.

- 화면 구조는 통일한다.
- 게임별 대표 색상과 아이콘으로 개성을 표현한다.
- 새 게임과 오늘의 퍼즐을 가장 중요한 행동으로 강조한다.
- 통계와 배지는 보조 행동으로 배치한다.
- 게임 소개와 규칙은 재사용 가능한 정보 카드로 제공한다.
- 장식은 풍부하게 사용하되 가독성을 해치지 않는다.
- 애니메이션은 짧고 기능적으로 사용한다.
- 모든 게임 화면은 공통 데이터 모델과 컴포넌트로 구현한다.
- 향후 게임 추가와 기능 확장을 고려한다.

최종적으로 사용자는 어떤 퍼즐을 선택하더라도 익숙한 구조 안에서 빠르게 게임을 시작할 수 있어야 하며, 동시에 각 게임만의 시각적 개성과 재미를 느낄 수 있어야 한다.
