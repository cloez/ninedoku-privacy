# K-퍼즐 세부 게임 공통 플레이 화면 디자인 가이드

## 1. 문서 목적

본 문서는 K-퍼즐에 포함된 여러 퍼즐게임의 실제 플레이 화면을 하나의 공통 디자인 시스템으로 구현하기 위한 가이드이다.

스도쿠 플레이 화면을 기준으로 정의한 상단 앱 바, 상태 정보, 퍼즐 보드, 도구 영역, 입력 영역의 구조를 유지하면서 각 게임의 규칙과 조작 방식에 맞게 변형할 수 있도록 설계한다.

대상 게임은 다음과 같다.

- 스도쿠
- 비나이로
- 지뢰찾기
- 음양
- 노노그램
- 킬러 스도쿠
- 스타 배틀
- 라이트업
- 후토시키
- 텐트
- 직소 스도쿠
- 빌딩
- 카쿠로
- 향후 추가되는 퍼즐게임

---

## 2. 설계 원칙

### 2.1 공통성과 개성의 균형

모든 게임은 동일한 앱 안에 있다는 느낌을 줘야 하지만, 게임의 규칙과 입력 방식까지 억지로 동일하게 만들면 안 된다.

다음 요소는 공통으로 유지한다.

- 상단 앱 바 위치
- 상태 정보 영역
- 진행 막대 스타일
- 보드 외곽 여백
- 도구 버튼의 모양
- 색상 토큰
- 타이포그래피
- 터치 피드백
- 완료 및 오류 애니메이션

다음 요소는 게임별로 변경한다.

- 보드 구조
- 셀 상태
- 입력 패드
- 도구 구성
- 상태 지표
- 대표 색상
- 힌트 표현 방식

### 2.2 플레이 집중 우선

게임 허브와 개별 게임 메인화면은 화려한 일러스트를 사용할 수 있지만, 실제 플레이 화면은 다음 기준을 우선한다.

1. 보드 가독성
2. 터치 정확성
3. 상태 구분
4. 입력 속도
5. 장시간 플레이 시 피로 최소화

### 2.3 데이터 기반 구성

게임별 화면을 각각 하드코딩하지 않고, 공통 레이아웃에 게임별 설정을 주입하는 구조를 권장한다.

## 4. 상단 앱 바

## 4.1 공통 구성

- 좌측: 뒤로가기
- 중앙: 모드명 또는 게임명
- 우측: 일시정지
- 선택 사항: 추가 메뉴

### 예시

```text
←        클래식        ||
```

## 4.2 모드명 예시

| 게임 | 모드명 예시 |
|---|---|
| 스도쿠 | 클래식 |
| 비나이로 | 8×8 |
| 지뢰찾기 | 초급 |
| 음양 | 클래식 |
| 노노그램 | 10×10 |
| 킬러 스도쿠 | 보통 |
| 스타 배틀 | 2스타 |

- 제목은 중앙 정렬
- 아이콘 버튼은 44×44dp 이상
- 앱 바는 보드보다 낮은 시각적 우선순위 유지
- 배경은 화이트 또는 투명
- 제목은 짙은 네이비 사용

| 카쿠로 | 난이도, 남은 칸, 실수, 시간 |

## 5.3 상태 아이콘 예시

- 난이도: 별, 게이지, 레벨 텍스트
- 남은 칸: 격자
- 실수: X
- 시간: 시계
- 남은 지뢰: 폭탄
- 깃발: 깃발
- 남은 별: 별
- 미조명 칸: 전구
- 남은 텐트: 텐트

## 6. 보드 공통 디자인

## 6.1 보드 컨테이너

- 화면 중심에 배치
- 배경은 화이트 또는 매우 연한 게임 테마색
- 바깥 프레임은 2~4dp
- 모서리는 12~20dp
- 보드 외부에 12~20dp 여백 확보
- 그림자는 최소화

## 6.2 셀 상태 토큰

| 상태 | 표현 |
|---|---|
| 기본 | 흰색 배경 |
| 선택 | 포인트색 테두리 + 연한 배경 |
| 관련 영역 | 매우 연한 테마색 |
| 동일 값 | 포인트색 숫자 또는 외곽선 |
| 사용자 입력 | 브랜드 블루 |
| 문제 제공 값 | 진한 네이비 |
| 메모 | 작은 회색 또는 라이트 블루 |
| 오류 | 연한 코랄 배경 + 경고 테두리 |
| 힌트 | 보라색 글로우 또는 점멸 1회 |
| 완료 | 부드러운 성공색 강조 |
| 비활성 | 낮은 채도 |

## 6.3 선 두께

- 일반 셀 선: 1dp
- 영역 구분선: 2~3dp
- 보드 외곽선: 2~4dp
- 선택 셀: 2dp

## 6.4 숫자 및 기호

- 문제 숫자: Bold
- 사용자 입력: Medium 또는 Bold
- 보조 힌트: Regular
- 숫자 크기: 셀 높이의 약 45~60%
- 메모 숫자: 셀 높이의 약 18~24%

## 7. 공통 도구 시스템

## 7.1 기본 도구 후보

- 되돌리기
- 다시 실행
- 삭제
- 메모
- 자동 메모
- 힌트
- 검사
- 초기화
- 표시 전환
- 깃발 모드
- 칠하기 모드
- X 표시
- 숫자 우선
- 셀 우선

## 7.2 도구 구성 원칙

- 한 화면에 4~6개 권장
- 6개를 넘는 경우 더보기 메뉴 사용
- 자주 사용하는 도구는 앞쪽 배치
- 활성 모드는 배경과 라벨을 함께 강조
- 비활성 기능은 투명도만 낮추지 말고 원인 안내 제공

## 7.3 공통 도구 표현

```text
[아이콘]
도구명
```

- 아이콘 버튼: 44~56dp
- 라벨: 12~15sp
- 도구 간격: 8~16dp
- 활성 배경: 연한 테마색
- 활성 아이콘: 진한 포인트색

# 9. 게임별 상세 적용 가이드

## 9.1 스도쿠

### 보드

- 9×9
- 3×3 영역 구분
- 행/열/같은 숫자 강조

### 입력

- 숫자 1~9
- 메모 모드
- 숫자 우선 모드

### 도구

- 되돌리기
- 삭제
- 메모
- 자동 메모
- 숫자 우선
- 힌트

### 특수 상태

- 동일 숫자 강조
- 충돌 숫자 경고
- 메모 자동 제거

---

## 9.2 비나이로

### 보드

- 6×6, 8×8, 10×10 등
- 각 셀에 0 또는 1
- 같은 숫자가 3개 연속되지 않도록 규칙 시각화

### 입력

- `0`
- `1`
- 삭제

### 도구

- 되돌리기
- 삭제
- 후보 표시
- 규칙 검사
- 힌트

### 특수 상태

- 연속 3개 충돌 행/열 강조
- 행 또는 열의 0/1 개수 초과 표시
- 중복 행/열 경고

### 테마

- 민트 + 네이비
- 0과 1을 서로 다른 채도와 모양으로 표현

---

## 9.3 지뢰찾기

### 보드

- 가변 크기 격자
- 닫힌 셀, 열린 셀, 깃발, 지뢰 상태 구분

### 입력

- 셀 열기
- 깃발 설치
- 길게 누르기로 깃발 전환 가능

### 도구

- 되돌리기
- 깃발 모드
- 안전 칸 힌트
- 확대/축소
- 일시정지

### 상태 정보

- 남은 지뢰
- 설치된 깃발
- 시간
- 난이도

### 특수 상태

- 지뢰 폭발: 짧은 코랄 강조
- 잘못된 깃발: X 표시
- 완료: 안전 칸 순차 애니메이션

### 테마

- 바이올렛 + 코랄
- 폭발 표현은 과도하지 않게 사용

---

## 9.4 음양

### 보드

- 흑/백 원형 또는 셀
- 연결 규칙과 2×2 동일색 금지 규칙 표현

### 입력

- 흑
- 백
- 빈칸

### 도구

- 되돌리기
- 삭제
- 자동 후보
- 연결 검사
- 힌트

### 특수 상태

- 2×2 동일색 충돌 표시
- 흑/백 연결이 끊긴 영역 강조
- 선택한 색과 동일한 셀 약하게 강조

### 테마

- 퍼플 + 화이트 + 차콜
- 흑백 대비는 충분히 확보

---

## 9.5 노노그램

### 보드

- 행/열 숫자 힌트
- 칠한 셀, X 셀, 빈 셀

### 입력

- 칠하기
- X 표시
- 지우기

### 도구

- 되돌리기
- 칠하기 모드
- X 모드
- 자동 완성 표시
- 힌트
- 확대/축소

### 특수 상태

- 완료된 힌트 숫자는 흐리게 표시
- 현재 행/열 강조
- 드래그 입력 지원
- 연속 드래그 시 같은 모드 유지

### 테마

- 오렌지 + 네이비
- 완성 그림을 방해하지 않는 단순한 셀 표현

---

## 9.6 킬러 스도쿠

### 보드

- 9×9
- 케이지 점선
- 케이지 합계 표시

### 입력

- 숫자 1~9
- 메모
- 숫자 우선

### 도구

- 되돌리기
- 삭제
- 메모
- 자동 메모
- 케이지 강조
- 힌트

### 특수 상태

- 선택 셀이 속한 케이지 강조
- 케이지 합계 초과 경고
- 완성된 케이지는 옅게 처리

### 테마

- 스카이블루 + 인디고

---

## 9.7 스타 배틀

### 보드

- 불규칙 영역
- 행/열/영역마다 정해진 개수의 별 배치

### 입력

- 별
- X 표시
- 지우기

### 도구

- 되돌리기
- 별 모드
- X 모드
- 자동 X
- 충돌 검사
- 힌트

### 특수 상태

- 별끼리 인접하면 경고
- 행/열/영역의 별 개수 표시
- 완료된 영역은 옅은 성공 표시

### 테마

- 골드 + 바이올렛

---

## 9.8 라이트업

### 보드

- 빈칸
- 검은 벽
- 숫자 벽
- 전구
- 조명된 셀

### 입력

- 전구
- X 표시
- 삭제

### 도구

- 되돌리기
- 전구 모드
- X 모드
- 미조명 셀 강조
- 충돌 검사
- 힌트

### 특수 상태

- 전구 빛이 닿는 셀을 연한 노랑으로 표시
- 전구끼리 서로 비추면 코랄 경고
- 숫자 벽 주변 전구 수 표시

### 테마

- 시안 + 옐로
- 조명 효과는 가독성을 해치지 않는 수준으로 사용

---

## 9.9 후토시키

### 보드

- 숫자 격자
- 셀 사이 부등호
- 라틴 방진 규칙 적용

### 입력

- 보드 크기에 맞는 숫자
- 메모

### 도구

- 되돌리기
- 삭제
- 메모
- 자동 메모
- 부등호 강조
- 힌트

### 특수 상태

# 23. 최종 디자인 원칙

K-퍼즐의 세부 게임 플레이 화면은 각기 다른 규칙을 가지고 있어도 다음 경험을 공통으로 제공해야 한다.

- 어디에서 뒤로 가는지 즉시 알 수 있다.
- 현재 난이도와 진행 상태를 빠르게 이해할 수 있다.
- 보드가 가장 잘 보인다.
- 입력 방식이 직관적이다.
- 활성화된 도구를 명확히 알 수 있다.
- 오류 이유를 이해할 수 있다.
- 힌트가 학습에 도움이 된다.
- 세로형과 가로형 모두 편리하다.
- 장시간 플레이해도 눈이 피로하지 않다.
- 게임 허브 및 메인화면과 같은 브랜드 경험을 유지한다.

게임마다 완전히 다른 UI를 만드는 것이 아니라, **공통 플레이 프레임 안에서 보드·입력·도구만 게임 규칙에 맞게 교체하는 구조**를 적용하는 것이 핵심이다.
