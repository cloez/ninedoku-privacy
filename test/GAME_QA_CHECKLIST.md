# 게임 공통 QA 테스트 체크리스트

> 모든 신규 게임은 이 체크리스트의 **전 항목**을 테스트해야 합니다.
> PM은 각 릴리스의 STEP 5(통합 QA)에서 이 체크리스트를 기준으로 검증합니다.
> QA가 PASS하지 않으면 STEP 6(챔피언 리뷰)으로 진행할 수 없습니다.

---

## A. 엔진 테스트 (최소 30개)

### A1. 보드 모델 (최소 5개)
- [ ] 빈 보드 생성 (기본 크기)
- [ ] getValue / setValue 동작 확인
- [ ] toJson / fromJson 라운드트립 정합성
- [ ] 고정(fixed) 셀 수정 불가 확인
- [ ] isComplete 판단 (전체 채움 + 규칙 충족)

### A2. 솔버 (최소 8개)
- [ ] 완성 보드의 isComplete == true
- [ ] 규칙 위반 보드의 isValid == false (각 규칙별 1개 이상)
- [ ] 빈 보드를 solve하면 유효한 해답 반환
- [ ] solve 결과가 isComplete == true
- [ ] hasUniqueSolution 검증 (유일해인 퍼즐)
- [ ] 해가 없는 모순 보드에서 solve == null
- [ ] solve가 원본 보드를 수정하지 않음 (불변성)
- [ ] hasUniqueSolution이 원본 보드를 수정하지 않음

### A3. 생성기 (최소 8개)
- [ ] 각 난이도별 퍼즐 생성 성공 (전 난이도 1개 이상)
- [ ] 생성된 퍼즐이 유일해를 가짐
- [ ] 같은 시드로 같은 퍼즐 생성 (결정성)
- [ ] 다른 시드로 다른 퍼즐 생성
- [ ] 생성 시간 3초 이내
- [ ] 100회 생성 시 모두 유일해 (복수 난이도 혼합)

### A4. 힌트 (최소 5개)
- [ ] Level 1: 대상 영역 인덱스 반환
- [ ] Level 2: 후보 값 반환
- [ ] Level 3: 풀이 기법 설명 문자열 반환
- [ ] Level 4: 정답 값 반환
- [ ] 힌트 결과가 실제 솔루션과 일치
- [ ] 힌트 설명에 숫자(0,1) 대신 게임 고유 기호(●,○ 등) 사용 확인

### A5. 통합 (최소 4개)
- [ ] 생성 → 풀이 → 완료 전체 사이클
- [ ] Board 직렬화 후 풀이 가능
- [ ] 빈 보드에서 힌트 요청 시 안전 처리

## B. 게임 상태 테스트 (최소 12개)

### B1. State 모델 (최소 7개)
- [ ] State 생성 및 기본값 확인
- [ ] copyWith 동작 (각 필드 독립 변경)

### B2. 등급 산정 (최소 5개)

- [ ] Excellent(A) 등급 조건
- [ ] Great(B) 등급 조건
- [ ] Good(C) 등급 조건
- [ ] 난이도별 기준 시간 값 확인

## C. 게임 로직(Notifier) 테스트 (최소 15개)

### C1. 기본 동작 (최소 6개)
- [ ] 초기 상태 null (SharedPreferences 없이 생성 시)
- [ ] startNewGame 시 상태 생성 확인
- [ ] startNewGame 시 보드 크기가 난이도와 일치
- [ ] pause / resume 동작
- [ ] giveUp: 상태 null + 저장 삭제
- [ ] hasOngoingGame 정확한 판단

### C2. 셀 입력 (최소 5개)
- [ ] 셀에 값 배치 (게임별 입력 방식)
- [ ] 같은 셀 재입력 시 토글/변경 동작
- [ ] 고정 셀 변경 불가
- [ ] 입력 모드 전환 (있는 경우)
- [ ] 지우기 동작

### C3. Undo (최소 2개)
- [ ] 마지막 동작 되돌리기
- [ ] 빈 스택에서 undo 호출 시 안전 처리

### C4. 완료 판정 (최소 2개)
- [ ] 모든 셀 정답 입력 시 isCompleted = true
- [ ] 완료 시 CompletedGameRecord 저장 확인 (SharedPreferences 키 확인)

## F. UI/UX 일관성 테스트 (수동 + 자동)

### F1. 화면 흐름 (수동 검증)
- [ ] 허브 → 게임 홈 → 모드/난이도 → 플레이 → 완료 → 결과 → 허브 (골든 패스)
- [ ] 게임 홈에서 이어하기 카드 표시 (진행 중 게임 있을 때)
- [ ] 게임 홈에서 규칙 설명 카드 표시 (빈 상태일 때)

### F2. 하드웨어 백키 (전 화면 필수 검증)
- [ ] 게임 홈: 백키 무시 (허브 이동은 아이콘으로만)
- [ ] 게임 플레이 중: 나가기 확인 다이얼로그 (AppBar 뒤로가기와 동일)
- [ ] 일시정지: resume()
- [ ] 결과 화면: giveUp + 게임 홈 이동

### F3. 가로 모드 (전 화면 검증)
- [ ] 게임 플레이: 보드 좌측 + 컨트롤 우측 레이아웃
- [ ] 가로 모드에서 AppBar 숨김 + 상단 바에 뒤로가기/게임명/일시정지
- [ ] 세로↔가로 전환 시 게임 상태 유지 (타이머, 입력값)
- [ ] 가로 모드에서 셀 크기 적절 (터치 가능)

### F4. 결과 화면 일관성
- [ ] 등급 표시 (S/A/B/C + 색상)
- [ ] 통계 표시 (시간, 난이도, 실수, 힌트)
- [ ] 버튼 순서: 새 게임(ElevatedButton, 상단) → 홈(OutlinedButton, 하단)
- [ ] 배지 획득 팝업 다이얼로그 (완료 직후 자동)
- [ ] 결과 화면 내 새 배지 칩 섹션

### F5. 통계/배지 접근
- [ ] 게임 홈에서 통계/배지 버튼 → 해당 게임 탭 자동 선택
- [ ] 허브에서 통계/배지 → 전체 탭
- [ ] 통계 화면에서 해당 게임 탭에 데이터 표시
- [ ] 배지 화면에서 해당 게임 탭에 배지 목록 표시

### F6. 다국어 (4개국어 전체)
- [ ] 게임 이름 표시 (한/영/일/중)
- [ ] 게임 설명 표시
- [ ] 규칙 설명 카드 (3가지 규칙 + 조작법)
- [ ] 힌트 메시지 (게임 고유 기호 사용, 숫자 아님)
- [ ] 배지 이름/설명
- [ ] 결과 화면 텍스트
- [ ] 일시정지/나가기/포기 다이얼로그

## 체크리스트 사용 방법

### 신규 게임 개발 시
1. 이 파일을 복사하여 `test/games/{게임명}/QA_CHECKLIST.md` 생성
2. 각 항목에 테스트 파일명과 테스트명 기록
3. 모든 항목에 ✅ 표시될 때까지 개발/수정 반복
4. PM이 STEP 5에서 체크리스트 검토

### 자동 테스트 파일 구조 (필수)
```
test/games/{게임명}/
  ├── {게임명}_engine_test.dart      ← 섹션 A (엔진)
  ├── {게임명}_state_test.dart       ← 섹션 B (상태)
  ├── {게임명}_notifier_test.dart    ← 섹션 C (로직)
  ├── {게임명}_storage_test.dart     ← 섹션 D (저장/복구)
  └── {게임명}_backup_test.dart      ← 섹션 E (백업)
```
섹션 F, G는 수동 검증 + 코드 리뷰로 대체 가능하나, 가능한 자동화 권장.

### 최소 테스트 수
| 섹션 | 최소 테스트 수 |
|------|--------------|
| A. 엔진 | 30개 |
| B. 상태 | 12개 |
| C. 로직 | 15개 |
| D. 저장/복구 | 8개 |
| E. 백업 | 6개 |
| **합계** | **71개 이상** |
